# Code Review

##  Fyra hattar

Ta på dig fyra olika hattar. Skriv kommentarer i koden, med förslag på förändringar, t.ex:

	// TODO: Metoden gör två saker och är för lång. Korta ner metoden
	// TODO: Om användaren matar in tom sträng så kraschar programmet
	// TODO: Denna metod används ej
	// TODO: Otydligt vad som händer här
	// TODO: Byt variabelnamn

Skriv positiva omdömmen, t.ex:

	// GOOD: Bra felhantering
	// GOOD: Snyggt grafiskt gränsnitt
	// GOOD: Tydligt flöde, jag fattar hur programmet funkar

Försök komma på så många kommentarer som möjligt. Tänk på att alla kommentarer kan hjälpa dina kollegor att göra ett bättre program.

##### Du är en kritisk användare
- Vilka fel kan du provocera fram?
- Kan du få programmet att krascha?
- Är det otydligt vad du som användare förväntas göra?


##### Du är en snäll användare
- Vad är bra ur ett användarperspektiv?
- Är det tydligt vad du förväntas göra?
- Är felhanteringen bra?
- Ser det grafiskt snyggt ut?


##### Du är en kritisk programmare. 
- Vilka fel hittar du i koden?
- Otydliga variablenamn, otydliga metodnamn
- För få eller för många kommentarer
- För långa metoder
- Gör en metod mer än en sak och kan styckas upp i flera?


##### Du är en snäll programmare. 
- Är det lätt att följa koden?
- Finns det smarta metoder?
- Ser du en genial lösning på ett svårt problem?
